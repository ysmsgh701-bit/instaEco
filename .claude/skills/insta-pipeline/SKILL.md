# 인스타 파이프라인 (오케스트레이터)

메인 세션이 이 스킬을 실행하며 오케스트레이터 역할을 한다. 4개 에이전트를 순서대로 호출하고, 각 단계 산출물을 게이트 검증한다. 게이트 실패 시 해당 에이전트에 SendMessage로 수정을 지시한다 (새 에이전트 생성 금지 — 컨텍스트 유지).

## 사전 준비

1. 오늘 날짜로 작업 폴더 결정: `posts/YYYY-MM-DD/`
2. 같은 날짜 폴더에 `04_final.md`가 이미 있으면 사용자에게 확인 후 진행.

## 실행 순서

### 1단계: 소주제 선정
- `insta-topic-scout` 호출 (run_in_background: false). 프롬프트에 작업 폴더 경로 포함.
- **게이트 1**: `01_topics.md` 존재 + 최종 선정 1개 + 후보 5개 전부 출처 URL 보유 + 최근 7일 주제와 중복 없음.

### 2단계: 컨텐츠 리서치
- `insta-content-researcher` 호출. 01_topics.md 경로 전달.
- **게이트 2**: `02_research.md` 존재 + 팩트 시트 5개 이상 + 핵심 수치 전부 교차확인 ✅ + 카드 구성안 존재.

### 3단계: 초안 작성
- `insta-writer` 호출. 02_research.md 경로 전달.
- **게이트 3**: `03_draft.md` 존재 + "재료 부족" 표기 없음 + 카드·캡션·해시태그 전부 존재. "재료 부족"이면 2단계 researcher에 보강 지시 후 3단계 재실행.

### 4단계: 검수·완성
- `insta-reviewer` 호출. 03_draft.md + 02_research.md 경로 전달.
- **게이트 4**: `04_final.md` 존재 + 판정 "통과" + 잔여 리스크 "없음". 반려(리서치 보강 필요)면 2단계로 복귀. 같은 단계 재시도 2회 초과 시 중단하고 사용자에게 보고.

## 최종 확정 (오케스트레이터 직접 수행)

04_final.md를 직접 읽고 마지막 점검:
1. 표지 훅이 스크롤을 멈출 만한가
2. 팩트 시트 대비 임의 추가된 수치가 없는가 (핵심 수치 2~3개 샘플 대조)
3. 계정 톤(2030 직장인 대상 해요체)과 일치하는가

이상 없으면 사용자에게 최종본 요약 보고:
- 주제 + 표지 훅
- 카드 수, 검수 결과
- `04_final.md` 파일 링크

이상 있으면 해당 단계 에이전트에 수정 지시 후 재검증.

## 보고 형식

각 단계 완료 시 한 줄 상태만 출력 (예: "1/4 소주제 선정 완료 — (주제)"). 상세 설명 금지. 최종 보고만 요약 포함.
